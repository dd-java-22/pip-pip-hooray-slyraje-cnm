## Basic User Instructions

### Launching and signing in

1. Install and launch the app on a supported device or emulator.
2. When prompted, complete **Google Sign‑In** using a valid Google account; sign‑in is required to reach the main screens.

### Creating an incubator

1. From the bottom navigation bar, tap **Incubator** to open the incubator list screen.
2. Tap the **floating action button (FAB)** to create a new incubator.
3. Enter the incubator **name**, **model**, and target **temperature** and **humidity** values.
4. Save the incubator; it will appear in the incubator list and be available for selection when creating batches.

### Creating a batch

1. From the bottom navigation, tap **Home** to open the batch list screen (the app’s main screen).
2. Tap the **FAB** on the batch list to create a new batch.
3. Enter the batch **number**, choose an **incubator**, specify one or more **breeds**, and set the **date set**.
4. Review the automatically populated candling and lockdown milestone dates; adjust them if needed.
5. Enter the **number of eggs set** and any relevant **notes**, then save the batch.
6. The new batch will appear as a card in the batch list on the Home screen.

### Viewing and updating batch details

1. On the Home screen, tap any batch card to open its **batch detail** view.
2. Review the batch metadata, including incubator, breeds, milestones, and notes.
3. Use the **bulk candling by group** action to record updated viability counts based on candling observations; the viability total for the batch will be updated accordingly.
4. The edit icon on this screen is currently not implemented and will display a toast message if tapped.

### Using settings

1. From the bottom navigation, tap **Settings** to open the settings screen.
2. Review the available options for incubator default temperature/humidity, temperature scale, notifications, CSV export, and sign‑out.
3. At present, most settings are placeholders and do not affect app behavior; the **Sign out** option is functional and will return you to the Google Sign‑In flow on next launch.
